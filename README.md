# StegShield

StegShield is a bachelor thesis project for detecting suspicious and dangerous image files using a hybrid approach:

- CNN-based image analysis.
- Metadata and file-structure inspection.
- A fusion layer that labels files as `safe`, `suspicious`, or `dangerous`.

The first version will be an installable command-line tool:

```bash
stegshield analyze ./image.png
```

Later versions will expose the same analysis pipeline through an API so applications can scan uploaded images, such as profile pictures or user-generated content.

## Current Status

The project has an initial metadata analyzer CLI and the first custom CNN training scaffold.

## Planned Features

- File type validation.
- Hashing and file metadata extraction.
- EXIF and structural risk indicators.
- Custom CNN trained from scratch.
- Optional comparison against pretrained CNN models.
- Combined CNN + metadata decision layer.
- CLI output in human-readable and JSON formats.
- Future FastAPI integration.

## Labels

- `safe`: no suspicious indicators.
- `suspicious`: weak or moderate risk indicators.
- `dangerous`: strong evidence of payload-like behavior, steganographic embedding, malformed structure, or multiple high-risk indicators.

## Development Setup

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -e ".[dev]"
```

For CNN training and evaluation, install the ML dependencies. On a local CPU machine, install PyTorch from the official CPU wheel index first, then install the project ML extra:

```powershell
python -m pip install -r requirements-ml-cpu.txt
python -m pip install -e ".[dev,ml]"
```

If you want CUDA/GPU support, use the install command generated by the official PyTorch selector for your CUDA version: https://pytorch.org/get-started/locally/

Run the placeholder CLI:

```bash
stegshield --help
```

Run tests:

```bash
pytest
```

## Dataset Preparation

Place images in the labeled folders:

```text
data/raw/safe/
data/raw/suspicious/
data/raw/dangerous/
```

Then create train/validation/test CSV files:

```bash
stegshield prepare-dataset
```

This writes:

```text
data/splits/train.csv
data/splits/val.csv
data/splits/test.csv
```

If the dataset already provides official train/validation/test folders, preserve those
splits instead of reshuffling them. The `prepare-dataset` command refuses to overwrite
existing split CSV files unless `--force` is passed.

For the Kaggle Stego Images Dataset, use the dedicated importer after organizing files
under `data/raw/{safe,suspicious,dangerous}/kaggle_stegoimages/`:

```powershell
stegshield import-kaggle-splits
```

This writes portable relative paths and preserves the dataset split structure:

```text
data/splits/train.csv
data/splits/val.csv
data/splits/test_standard.csv
data/splits/test_adversarial.csv
data/splits/test_full.csv
```

Use `test_standard.csv` for normal comparable evaluation, `test_adversarial.csv` for
extra encoded/archive stego variants, and `test_full.csv` only when you want both
groups together.

## Custom CNN Training

After installing ML dependencies and preparing the dataset:

```bash
stegshield train-cnn --epochs 5 --batch-size 16 --image-size 256 --device cpu
```

The image loader crops by default instead of downscaling, because resizing can destroy
pixel-level steganography artifacts.

The default outputs are:

```text
outputs/models/stegshield_cnn.pt
outputs/reports/training_metrics.json
```

## Dataset Direction

Candidate datasets include:

- Kaggle Stego Images Dataset: https://www.kaggle.com/datasets/marcozuppelli/stegoimagesdataset
- StegoAppDB: https://forensicstats.org/stegoappdb/
- BOSSBase / BOWS2-based steganalysis datasets.
- ALASKA steganalysis datasets.

Real malicious files should not be executed. Suspicious samples must be handled defensively and, if necessary, in an isolated environment.
